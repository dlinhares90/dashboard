⚙️ Como o Projeto Funciona (passo a passo)

1. Lead entra no Kommo

Alguém clica no anúncio do Instagram/Meta → vira um lead automaticamente no funil "Funil de Vendas" do Kommo CRM, na etapa Leads de entrada.

2. CRC entra em contato

A equipe CRC (Central de Relacionamento com o Cliente) liga ou manda mensagem para tentar agendar uma consulta. O lead avança de Novo Lead → Cadência ligações → Follow Agendamento.

3. Lead é agendado

Se o lead aceita, vai para Agendado. Se faltou na consulta, vai para Follow Up No Show para re-agendar.

4. Consulta [PT] — avaliação presencial

O paciente vai à clínica. O dentista/avaliador faz a consulta e, se tudo correr bem, passa para Proposta Feita.

5. Fechamento

Paciente aceita a proposta → Ganha ✅. Não aceita → Perdida ❌. Precisa de exames antes → Controle de Exames.

6. Você (CS) acompanha tudo isso

Roda python server.py localmente, acessa o dashboard, e toda semana gera um relatório para enviar ao Arthur com diagnóstico e próximos passos.

📖 Glossário — Termos do Projeto
Kommo CRM

Software de gestão de clientes (CRM = Customer Relationship Management). É onde ficam todos os leads, tarefas e histórico de contato da clínica. Acessado em arthurdiasimplantodontista.kommo.com.

Lead

Pessoa que demonstrou interesse (clicou no anúncio, mandou mensagem). Ainda não é paciente — é um potencial cliente que precisa ser convertido.

Funil de vendas

Representação visual da jornada do lead: começa com muitos no topo (entrada) e vai afunilando até os poucos que fecham (ganhos). Cada etapa tem um nome e um ID numérico no Kommo.

CRC

Central de Relacionamento com o Cliente. Equipe responsável por fazer o primeiro contato com o lead — ligações, WhatsApp — e tentar agendar a consulta.

CS

Customer Success. Seu papel: garantir que o cliente (Arthur Dias) esteja usando o Kommo corretamente, atingindo as metas do playbook e crescendo. Você não vende — você garante o sucesso da operação.

API do Kommo

Interface técnica que permite buscar dados do Kommo de forma automática (sem precisar abrir o site). É como uma "tomada de dados" — você conecta um script Python nela e puxa os números. Requer um token de autenticação (guardado no .env).

Token / .env

O token é uma senha longa que autoriza seu script a acessar a API do Kommo. O arquivo .env é onde ele fica guardado localmente — nunca sobe para o GitHub para não expor a senha publicamente.

CORS

Política de segurança do navegador que bloqueia chamadas a APIs externas feitas diretamente pelo JavaScript de uma página. Por isso existe o server.py — ele age como intermediário (proxy) entre o dashboard e a API do Kommo.

GitHub Pages

Serviço gratuito do GitHub que publica arquivos HTML/CSS/JS como um site na internet. Só funciona com arquivos estáticos (sem servidor). Por isso o relatório funciona aqui mas o dashboard não.

Playbook

Documento interno da Power Mídias com as regras, metas, scripts e processos que devem ser seguidos pelo CS e pela equipe do cliente. É a "bíblia" da operação.
